# I2C Multi-Sensor Driver for STM32G474RE

Drivers for 5 I2C sensors on NUCLEO-G474RE development board, covering three I2C communication patterns.

## Hardware

- **MCU**: STM32G474RE (NUCLEO-G474RE)
- **I2C Bus**: I2C1 — PB8(SCL), PB9(SDA)
- **Serial**: USART2 via ST-Link VCP — COM10, 115200 baud

| Sensor | Chip | Address | Type | Pattern |
|--------|------|:-------:|------|---------|
| Barometer | SPL06-001 | 0x77 | Pressure + Temperature | Register-based |
| Temp/Humidity | AHT20 | 0x38 | Temperature + Humidity | Raw I2C |
| Color | BH1745 | 0x39 | RGB + Clear | Register-based |
| IMU | QMI8658C | 0x6A | 6-Axis Accel + Gyro | Register-based |
| ToF Distance | VL53L0X | 0x29 | Laser Ranging | API + Platform Layer |

## Three I2C Patterns Learned

| Pattern | Sensors | HAL Function |
|---------|---------|-------------|
| **Register-based** | SPL06, BH1745, QMI8658C | `HAL_I2C_Mem_Read/Write` |
| **Raw I2C** | AHT20 | `HAL_I2C_Master_Transmit/Receive` |
| **API + Platform Layer** | VL53L0X | Custom `vl53l0x_hal.c` adapter |

## Build

- **Toolchain**: `arm-none-eabi-gcc` from STM32CubeCLT 1.22.0
- **Build system**: CMake (generated by CubeMX)
- **IDE**: CLion

```bash
# Compile
cmake --build build/Debug --target IIC_test -j

# Flash (STM32CubeProgrammer CLI)
STM32_Programmer_CLI -c port=SWD -w build/Debug/IIC_test.elf -rst

# Serial monitor
python -c "import serial; s=serial.Serial('COM10',115200,timeout=5); print(s.read(4096).decode())"
```

## Key Lessons

- **BOOT0 conflict**: PB8 doubles as BOOT0 pin. Sensor pull-up can cause boot from system memory. Fix with Option Bytes: `nSWBOOT0=0 nBOOT0=1`
- **Gyro zero-rate offset**: MEMS gyros have ±10 dps offset at rest. Calibrate by averaging 100 samples on startup.
- **VL53L0X architecture**: Contains internal MCU running firmware. Requires ST's API — cannot be driven by raw register writes.
- **Platform layer pattern**: ST API declares functions in `.h` files; your `.c` file provides the implementation. The linker connects them at build time.
